metrics-server 설치

metrics-server (opens in a new tab) 는 k8s 플랫폼마다 기본으로 설치된 곳도 있고 아닌 플랫폼도 있습니다. 로컬 클러스터인 kind 클러스터 (opens in a new tab)에서도 설치가 되어있는지 확인이 필요합니다.

클러스터 내에 metrics-server가 설치되었는지 확인을 합니다.

bash
$ kubectl top nodes
error: Metrics API not available

metrics-server 가 설치되어 있지 않습니다. metrics-server (opens in a new tab) 에 방문해서 metrics-server를 설치합니다.

metrics-server#installation (opens in a new tab)에서는 metrics-server 를 설치하는 명령어를 알려주고 있습니다. 이 명령어를 복사해서 쿠버네티스 클러스터에 적용해줍니다.

bash
kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

이렇게 설치를 해도 아래와 같이 kubectl top nodes 로 현황을 확인해도 에러 메시지가 나타나는 경우가 있습니다.

bash
$ kubectl top nodes
error: Metrics API not available

spring boot 톰캣 스레드 풀 사이즈 수정

스프링 컨테이너의 커넥션을 처리하는 스레드가 부족해지는 상황을 가정하기 위해 스프링 부트 톰캣 스레드 풀의 사이즈를 10으로 수정합니다. 스프링 부트 톰캣 스레드 풀의 기본사이즈는 200 입니다.


requests, limits 수정

파드의 스펙을 아래와 같이 수정해줍니다.

  • memory
    • requests : "512Mi"
    • limits : "1Gi"
  • cpu
    • requests: "1000m"
    • limits: "1500m"
